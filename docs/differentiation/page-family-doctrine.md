# SideGuy Page Family Doctrine

Purpose:

Scale SideGuy with differentiated content families instead of duplicate boilerplate pages.

Every topic cluster should include distinct page types:

1. explainer
2. cost
3. comparison
4. industry
5. local

Each type must use different required sections.

--------------------------------------------------

EXPLAINER PAGE BLOCKS
- what it is
- how it works
- who it helps
- common misconceptions
- related concepts

COST PAGE BLOCKS
- pricing models
- cost drivers
- hidden costs
- ROI considerations
- when cheaper alternatives are enough

COMPARISON PAGE BLOCKS
- where option A wins
- where option B wins
- tradeoffs
- implementation complexity
- SideGuy take

INDUSTRY PAGE BLOCKS
- industry workflow examples
- operator pain points
- common use cases
- integration notes
- what to avoid

LOCAL PAGE BLOCKS
- local context
- regional costs
- local operator considerations
- implementation notes
- local CTA

--------------------------------------------------

RULE

Do not publish pages that reuse the same body copy across multiple intents.

Same topic is fine.
Same template body is not.

Goal:

Build topic authority with semantic differentiation.
