# SideGuy Clarity Pyramid Architecture

The internet is noisy.

SideGuy organizes knowledge using a clarity pyramid.

Base → Systems → Decisions

This structure keeps content helpful, human, and actionable.

---

Base Layer — Real Problems

Examples:

why-is-my-ac-blowing-warm-air  
how-do-machine-payments-work  
can-ai-agents-pay-for-apis  

Purpose:

capture real search intent.

Traits:

clear question  
clear explanation  
links to system pages  

---

Middle Layer — Systems

Examples:

machine-to-machine-payments  
stablecoin-settlement  
ai-automation-workflows  

Purpose:

explain the system architecture.

Traits:

educational  
infrastructure explanations  
tool ecosystems  

---

Top Layer — Human Resolution

Examples:

repair-vs-replace-hvac  
choosing-payment-systems  
ai-vs-human-support  

Purpose:

guide decisions.

Traits:

calm guidance  
decision frameworks  
human trust  

---

SideGuy Rule

AI explains systems.  
Humans resolve decisions.  
Technology executes actions.

---

Mission

SideGuy is where Google discovers the problem,  
AI explains the system,  
and a real human resolves the decision.
