# SideGuy Semantic Topic Lattice

Purpose:

Scale SideGuy toward 100,000 pages without becoming a pile of disconnected programmatic pages.

The lattice model organizes content into:

- parent topics
- child topics
- problem pages
- cost pages
- comparison pages
- glossary pages
- local overlays
- industry overlays

Each page must belong to a topic family.

--------------------------------------------------

LATTICE PRINCIPLES

1. Every page belongs to a parent topic.
2. Every parent topic has child topics.
3. Every child topic can generate:
   - what-is pages
   - how-to pages
   - cost pages
   - comparison pages
   - local pages
   - industry pages
4. Each page links upward to the parent hub.
5. Each page links sideways to sibling pages.
6. Each page links downward when deeper pages exist.

--------------------------------------------------

EXAMPLE

Parent topic:
AI automation

Child topics:
AI scheduling
AI customer support
AI operations
AI agents
workflow automation

Possible pages:
what-is-ai-scheduling.html
how-to-use-ai-scheduling-for-restaurants.html
how-much-does-ai-customer-support-cost.html
ai-agents-vs-workflow-automation.html
ai-operations-for-contractors-san-diego.html

--------------------------------------------------

QUALITY RULES

Every page must include:

- title
- meta description
- canonical
- H1
- intro paragraph
- at least 3 structured sections
- Text PJ orb
- Back to Home link
- command center link
- relevant hub link
- related guides block

--------------------------------------------------

GOAL

Build a knowledge network, not a spam farm.
