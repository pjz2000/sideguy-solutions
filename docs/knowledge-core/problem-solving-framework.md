# SideGuy Problem Solving Framework

Step 1 — Identify the real problem

People often search for symptoms rather than root causes.

Step 2 — Explain the system

Clarify how the underlying technology or market works.

Step 3 — Compare options

Present realistic trade-offs between solutions.

Step 4 — Provide guidance

Help the user determine the best next step.

Step 5 — Execute

Connect the user to the right operator, contractor,
or solution.

This is the SideGuy workflow.
