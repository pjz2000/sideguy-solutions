# SideGuy Wiki Graph Doctrine

The SideGuy Wiki is not just pages.

It is a graph of relationships between:

- problems
- systems
- tools
- decisions
- human operators

Every page should connect to other pages through meaningful relationships.

---

## Relationship types

Problem → System
Problem → Resolution
System → Tool
System → Problem
Resolution → Operator
Resolution → System
Skill → Operator
Category → All pages in cluster

---

## Why this matters

Search engines and AI systems understand knowledge better when content is structured.

SideGuy pages should not be isolated.

They should form a network.

---

## Example cluster

Problem:
can-ai-agents-pay-for-apis

---

## SideGuy graph rule

Every page must connect to:

1 parent hub  
2 related pages  
1 system explanation  
1 decision or resolution page  

This prevents orphan pages and strengthens topical authority.

---

## SideGuy graph mantra

Pages create information.

Links create knowledge.
