# Pretext Software Development

Before software is built, truth must be discovered.

## Doctrine
Cloud software is cheap to build.
Wrong assumptions are expensive.

SideGuy runs a Pretext phase before development:
- listen to Google demand
- map workflow friction
- surface hidden APIs and system layers
- identify customer confusion
- cluster commercial-intent searches
- discover what humans would not ask upfront
- define what should NOT be built
- prioritize the highest ROI system layer

## Promise
Before you spend $50k building software, spend a couple hundred dollars to make sure the build is intelligent.

## Product
- Day 1 signal onboarding
- 30–60 day intelligence report
- workflow + market truth
- optional Kromeon / SideGuy cloud build after validation
