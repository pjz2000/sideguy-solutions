# SidePalantir Operating System

## Purpose
SidePalantir is the internal intelligence layer for SideGuy Solutions.

It exists to:
- watch meaningful internet and business signals
- detect future-relevant topics early
- prioritize the best pages to improve
- identify missing hub coverage
- route signal into useful SideGuy content
- support the human resolution layer via Text PJ

## Core Principle
Watch the signal.
Ignore the noise.
Build where trust compounds.

## Signal Categories
- AI automation
- payment infrastructure
- stablecoins
- instant settlement
- machine-to-machine commerce
- local operator pain points
- robotics
- EV / energy shifts
- business software confusion
- future infrastructure

## SideGuy Filter
A signal is worth acting on if it is:
1. useful to a real human
2. explainable clearly
3. likely to generate search demand
4. aligned with future infrastructure
5. compatible with SideGuy's calm human-first brand

## Output Types
Signals can become:
- new SEO page
- page upgrade
- hub expansion
- FAQ block
- meme page
- operator note
- future reserve topic

## Daily Operator Loop
1. Run trend-radar.sh
2. Run opportunity-score.sh
3. Run hub-gap-detector.sh
4. Pick top 1-3 actions
5. Build or improve
6. Log result
7. Repeat

## Brand Doctrine
Google discovers the problem.
AI explains the problem.
A real human resolves it.

Clarity before cost.
