## Non-negotiable internal linking rules

### Every LEAF page has:
- Back to Cluster
- Back to Pillar (and optionally Category)
- Related Problems (same cluster)
- See also (related clusters)
- Text PJ CTA

### Every CLUSTER hub has:
- What it is (friendly)
- Common misunderstandings
- Leaf list (or "Top leaves")
- See also (related clusters)

### Every CATEGORY hub has:
- A curated set of clusters
- "Start here" explanation
- Cross-links to related categories

### Every PILLAR hub has:
- Category list + cluster list
- Featured leaf problems (best entry points)

## The "Gravity Engine" principle

Backlinks + traffic land anywhere.
Your internal system should **spread authority** to everything else:
Pillar → Category → Cluster → Leaf
and sideways across clusters (topic graph).

## Traffic Intelligence loop (the compounding flywheel)

1. Export GSC queries/pages weekly
2. Find clusters gaining impressions (even with low clicks)
3. Expand those clusters:
   - 1 cluster hub upgrade
   - +5 to +50 new leaves
   - add internal links + see-also
4. Repeat forever

## What NOT to do

- Don't "just add more leaves" blindly once you have inventory
- Don't create orphan pages
- Don't let the site become a pile of unrelated URLs
- Don't ignore navigation pages (directories/categories) — they are the crawl highways
