# SideGuy Trend Radar Doctrine

## Purpose
Turn fresh internet developments into practical SideGuy page upgrades.

## Workflow
1. Scan recent feeds
2. Detect SideGuy-relevant themes
3. Match themes to existing inventory
4. Upgrade existing pages first
5. Create new pages only where signal has no good home
6. Strengthen internal links back into hubs

## Priority rules
- Upgrade strong relevant pages before making random new ones
- Improve pages with money intent, hub authority, and local trust first
- Add freshness, FAQs, process blocks, examples, and the best Text PJ orb
- Keep SideGuy calm, useful, and human-first

## Required upgrade blocks
- freshness signal
- FAQ
- how it works / process
- example or scenario
- contextual internal links
- visible human CTA
- local clarity-before-cost framing

## Canonical mantra
Fresh signals in.
Better pages out.
Authority compounds.
